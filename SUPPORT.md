# Support

Thanks for using an Eunolabs project. Here is where to go for help.

## Before you ask

- **Read the project's `README.md` and its `docs/`.** Most usage questions —
  installation, configuration, and common workflows — are answered there.
- **Search existing issues and discussions.** Someone may have already hit the
  same thing.

## Where to go

| I want to… | Go to |
| --- | --- |
| Ask a usage question or share an idea | The repository's **Discussions** tab, if enabled; otherwise open an issue. |
| Report a bug | Open a **bug report** issue in the relevant repository. |
| Request a feature | Open a **feature request** issue in the relevant repository. |
| Report a security vulnerability | **Do not open an issue** — follow [`SECURITY.md`](./SECURITY.md). |

## Writing a good request

The faster we (or another community member) can reproduce your problem, the
faster it gets resolved. Please include:

- The project and its version, release tag, or commit SHA.
- Your operating system and architecture.
- What you did, what you expected, and what actually happened.
- A minimal reproduction and any relevant logs or output — with secrets
  redacted.

## Response expectations

These are open-source projects maintained on a best-effort basis. Maintainers
triage issues and discussions as time allows; there is no guaranteed response
time or commercial support channel. Clear, reproducible reports get attention
soonest, and well-scoped pull requests are always welcome — see
[`CONTRIBUTING.md`](./CONTRIBUTING.md).
